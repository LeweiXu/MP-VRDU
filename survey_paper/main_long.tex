\section{Framework Architecture Overview and Trends}
Previous VRDU surveys categorise models by OCR dependency, separating OCR-based models that consume text and layout boxes from OCR-free models that consume document images directly \cite{huang2022layoutlmv3, ding2025survey}. This taxonomy is less informative in the multi-page setting, as OCR-using models often use OCR'd text as just one component in a multimodal pipeline \cite{sun2025docagent, wang2025vidorag}, while OCR-free models are often pretrained or aligned with OCR-derived supervision \cite{tanaka2025vdocrag, hu2024mplug}. The defining problem in MP-VRDU is how a system controls document-scale evidence while bounded by context window limits. Relevant evidence may be sparse, distributed, and multimodal, models must decide what information to retain and how it should interact across pages. Architectural design determines how evidence is selected, preserved, aggregated, and exposed for reasoning. We therefore organise multi-page frameworks primary by architectural trend, with OCR dependency as a secondary descriptor.

% \subsection{OCR Dependency}
% Earlier VRDU work typically separates OCR-dependent and OCR-free models according to whether they consume OCR tokens and layout boxes or read document images directly \cite{huang2022layoutlmv3, ding2025survey}. In MP-VRDU, this boundary becomes less binary, as OCR is often only one signal within a larger document-level system. It can support sparse retrieval, page grounding, bounding-box references, and textual verification, while visual encoders and VLMs handle image-based evidence \cite{han2025mdocagent, zhu2025simple, wang2025vidorag}. OCR remains valuable as structured text and layout coordinates provide compact representations for text-heavy or low-resolution pages; for example, LayTokenLLM projects OCR tokens, bounding boxes, and layout cues into the language-model space for efficient document reasoning \cite{zhu2025simple}. However, this compactness comes with OCR error propagation and parser dependence \cite{liao2025doclayllm}, and multi-page OCR-based systems still require retrieval, compression, or aggregation mechanisms to control document length \cite{hannan2025docslm, zhang2024cream}.\cite{hannan2025docslm, zhang2024cream}.

\subsection{Page-to-Document Transformer}
% Cut out specific techniques, just talk about 1. definition/characteristics 2. advantages (specific cases/domains) 3. limitations/disadvantages
This family extends single-page document modelling to the multi-page setting while preserving explicit cross-page interaction. Pages are encoded separately through a learned document encoder, with a document-level information exchange mechanism added for cross-page understanding. This architecture provides a strong inductive bias for multi-page structure, as page identity, local layout, and document order can be retained while evidence is aggregated across the document. Explicit cross-page modelling makes these models well suited to tasks where relevant evidence appears across pages within a coherent document structure, such as reports and business documents \cite{tito2023hierarchical, borchmann2025arctic}. Its limitations follow from the same design. Computation grows with document length, and all pages typically need to be encoded, making very long documents difficult to scale. Document-level representations may also lose fine-grained evidence when page information is compressed or weakly connected \cite{blau2024gram, dong2024multi}. 

% Rather than flattening all tokens and visual patches into one sequence, these models preserve page boundaries while connecting pages through summary tokens \cite{tito2023hierarchical}, global document tokens with local-global attention \cite{blau2024gram}, chunked sparse attention over long OCR streams \cite{borchmann2025arctic}, or recurrent memory passed across pages \cite{dong2024multi}. This design provides an inductive bias for multi-page understanding as evidence can be aggregated across pages without discarding page identity. However, cross-page reasoning is still bounded by the exchange mechanism, as summaries and memories can lose fine-grained evidence, sparse attention may miss distant dependencies, and computation generally grows with document length.

% This family uses a learned document encoder to separately encode individual pages, then introduces a document-level mechanism for information exchange across page representations. Hi-VT5 encodes each page independently and passes question-conditioned page tokens to the decoder, but its cross-page reasoning is delayed until after page-level compression \cite{tito2023hierarchical}. GRAM addresses this bottleneck by injecting document-global tokens into the encoder so pages can exchange information during representation learning, although its cost still increases with document length and optional compression can reduce answer quality \cite{blau2024gram}. RM-T5 instead carries recurrent memory tokens from page to page, making document aggregation more parameter-efficient, but fixed sequential memory can over-compress or forget earlier evidence \cite{dong2024multi}. Arctic-TILT pushes the same direction to much longer OCR-based inputs through layer-wise multimodal fusion and chunked sparse attention, but remains sensitive to OCR quality, visual layout complexity, and position bias in very long documents \cite{borchmann2025arctic}. These architectures preserve page structure and make document-level interaction explicit, but reasoning over many pages must still pass through compressed page summaries, memory states, or sparse attention patterns.

\subsection{Backbone-Centric MLLM Adaptation}
% For this part, we can emphasie the key point is continue pretraining the MLLM backbone. Then, you can introduce the frameworks from training paradigms updates (full training to parameter efficient tuning) and introducing additional machemisms. 

This family adapts a pretrained MLLM backbone to multi-page document understanding without introducing a separate document-level pipeline or agentic loop. Modifications are concentrated in the input pathway and lightweight tuning components, where page images, OCR text, and layout signals are projected into the backbone's token space and consumed as a single multi-page sequence \cite{zhu2025simple}. This design inherits the instruction-following, multimodal, and broad-domain reasoning capabilities of the underlying foundation model, and adaption can be parameter efficient as most components remain frozen while only projectors, adaptors, or compression modules are trained. These models are well suited to tasks that combine document reading with open-ended reasoning and instruction following, such as document-grounded dialogue and mixed domain question answering \cite{tanaka2024instructdoc, duan2025docopilot}, and moderate length documents where long context tuning closes the gap between single-page models and full retrieval pipelines \cite{hu2024mplug, yu2024texthawk2}. However, performance degrades beyond moderate-length documents, as the backbone must absorb document-scale evidence internally, quality remains dependent on context length and the quality of fine-grained text and visual representations after token compression is applied to fit pages into context \cite{jia2024leopard}. Reasoning over visual elements such as charts, tables, and figures depends entirely on the backbone's pretrained visual capabilities, and as many models in this family are OCR-free, reliable text recovery further requires high-resolution page images that aggressive downsampling readily compromises. Document structure such as section hierarchy and figure-caption linkages must likewise be recovered implicitly from pixels rather than supplied through layout tokens or a structural encoder \cite{hannan2025docslm, duan2025docopilot}

% Backbone-adapted MP-VRDU methods extend pretrained MLLMs to support multi-page document understanding by modifying the model architecture, training strategy, or input representation. These methods aim to encode page images, OCR text, and layout information into a unified document-level representation. Early approaches mainly conducting continuing pretraining or instruction tuning on projectors or lightweight adapters to align document-specific visual and textual signals with pretrained MLLMs \cite{jia2024leopard, tanaka2024instructdoc}. Subsequent work places greater emphasis on efficient document representation, introducing visual token compression \cite{}, layout-aware encoding, streaming inference, or long-context training to reduce the computational cost of multi-page inputs \cite{hu2025mplug, hannan2025docslm, yu2024texthawk2}. More recent studies further enhance document-level reasoning through large-scale multi-page supervision or evidence-guided reinforcement learning \cite{duan2025docopilot, xiong2026docr1}. Overall, this paradigm preserves the general instruction-following and multimodal reasoning capabilities of foundation models while adapting them to the structural and evidential demands of long documents. Its main limitation is that critical evidence must still be encoded and retained within the backbone, making performance highly dependent on context length, compression quality, OCR–vision alignment, and the preservation of fine-grained cross-page information.

\subsection{Retriever-Generator}
% For this part you can describe based on retrieving content (granuliary), retrieving methods (dense, sparse, joint, grep) and then emphasing the correlation between chunks. 

This family decomposes MP-VRDU into evidence selection and answer generation in which a retrieval module first identifies a compact subset of relevant document content, and a generator answers using only the selected evidence. Retrieved units may be OCR chunks, pages, visual patches, multmimodal elements, or graph nodes, and the two stages are typically developed and tuned independently. Inference cost depends on the amount of retrieved content rather than document length, allowing this design to scale to deocuments that exceed any current MLLM context window. Modular decomposition allows the retriever and generator to be swapped independently while retrieved evidence provides a partial audit trail for the answer. This same retrieval paradigm also extends naturally from a single long document to a corpus, supporting multi-document question answering without significant changes in architecture. As such, these models are well suited to very long documents and to structured domains such as scientific papers and technical manuals, where headings and sections provide reliable retrieval anchors \cite{cho2024m3docrag, shin2025multidocfusion}. Visually rich pages further benefit when when page-level visual retrieval preserves figures, tables, and layout that OCR-only pipelines lose \cite{tanaka2025vdocrag, xie2024wukong}. Limitations follow from the same single-pass decomposition design, as evidence missed at the retrieval stage is unrecoverable at generation time, and adding more retrieved context to compensate introduces noise and attention dilution. Performance also depends on parser and embedding quality, as chunking and OCR errors, as well as modality mismatches between retriever and generator propagate directly into the answer. Cross-page reasoning is further constrained by the assumption that relevant evidence can be ranked by query similarity, which weakens when answers require aggregating evidence that are not individually strongly relevant to the question \cite{wang2024knowledge, wu2025molorag}.

% This family decomposes MP-VRDU into evidence selection and answer generation in which a retrieval module first identifies a compact subset of relevant document content, and a generator answers using only the selected evidence. Retrieved units may be OCR chunks, pages, visual patches, multmimodal elements, or graph nodes, depending on whether the document is represented as text, images, interleaved content, or a structured evidence graph. Retrieval can be performed through course-to-fine selection \cite{zhang2024cream}, adaptive visual retrieval \cite{li2025avir}, multimodal dense retrieval \cite{cho2024m3docrag, tanaka2025vdocrag}, end-to-end sparse sampling over paragraphs and figures \cite{xie2024wukong}, graph-based prompting \cite{wang2024knowledge}, or logic-aware retrieval \cite{wu2025molorag}. This architecture scales well by preventing the generator from processing the full document, while the selected evidence provides partial interpretability. However, this introduces a retrieval bottleneck in which missed evidence is usually unrecoverable at generation time, while adding more retrieved context can introduce noise or attention dilution.

\subsection{Agentic Pipeline}

Recent surveys characterise agentic LLM systems through their capacity to reason, act, and interact with an external environment beyond a single forward pass \cite{plaat2025agentic, sapkota2025aiagents}. Following this framing, we categorise MP-VRDU models that performs multi-pass evidence acquisition or distributes work across specialised agents as agentic, while reserving the retriever-generator family for single-pass pipelines. Agentic systems thus treat MP-VRDU as an iterative evidence-seeking process, where a controller MLLM decides what to inspect next based on intermediate observations, issuing tool calls for retrieval, page navigation, region cropping, or OCR, and may distribute work across agents responsible for retrieval, visual inspection, synthesis, or adjudication. Iteration removes the single-pass bottleneck of retriever-generator systems, as the controller can revise its query or fetch additional evidence after observing partial results. Role specialisation tunes prompts to each subtask, inference compute scales with question difficulty rather than document length, and each step in the trajectory is inspectable, forming a more comprehensive audit trail. This design is well suited to questions whose evidence is distributed across pages or modalities, where a single retrieval pass is unlikely to surface everything required, and to multi-hop questions that need explicit query reformulation between steps \cite{wang2025vidorag, sun2025docagent}. Visually heterogeneous documents further benefit, as text, tables, and figures can be inspected with different tools and aggregated by the controller \cite{han2025mdocagent, zhu2025doclens}. Its limitations follow from the same iterative design. Each round invokes one or more frozen models and the number of rounds is question-dependent and difficult to bound in advance, so latency grows with task complexity, while action policies, role definitions, and stopping criteria are sensitive to prompt phrasing and prone to silent failure when the controller commits early to wrong evidence. Reproducibility and evaluation are further complicated by stochastic generation and the lack of a single forward pass to attribute answers to, and the models inherits any errors of the tools it calls, since iteration can only revisit content the underlying retrievers, parsers, and OCR modules can reach \cite{wang2025vidorag, sun2025docagent}.

% This family treats MP-VRDU as an iterative evidence seeking process rather than a fixed retrieve-then-generate pass. In this survey, this category refers to systems that use multi-agent decomposition, tool use, memory, or open-ended control to navigate long documents, inspect evidence, refine queries, and revise answers. Some pipelines assign retrieval, visual inspection, modal fusion, judgement and answer synthesis across specialised agents \cite{han2025mdocagent, duan2025m2rag, yu2025visual}, or use tool-augment and dynamic iterative workflows to navigate documents, inspect evidence, maintain intermediate states, and revise answers \cite{sun2025docagent, zhu2025doclens, zheng2026doc, wang2025vidorag}. This design is effective when evidence is scattered, visually fine-grained, or missed by a single retrieval step, allowing the model to gather additional context before answer generation. However, this introduces higher latency, repeated model calls, dependence on external parsing or tool quality, and greater difficulty in reproducing and evaluating the full decision process.

\section{Modalities Overview and Trends}
% MP-VRDU models must model text, visual elements, and layout over multiple pages. Hierarchical models usually encode modalities locally within each page before document-level aggregation. Backbone-centric MLLMs project or compress multimodal page evidence into the backbone context. Retrieval-generator and agentic pipeline models convert modalities into searchable or navigable evidence units.

\subsection{Text Modality}
In MP-VRDU, text remains a compact, discrete, and searchable representation of document content that carries much of the evidence questions depend on. It is typically obtained through OCR or PDF parsing, or recovered directly from page pixels by an LVLM. When extracted via OCR or parsing, its compactness enables indexing and retrieval over documents that exceed current context windows, supports natural answer-passage attribution, and provides a bridge to visual elements through captions, headings, and references that locate corresponding tables and figures. These advantages come at the cost of OCR and parser error propagation, fragile multi-column and table reading order, and the loss of visual semantics carried by charts, layout, or handwriting once a document is reduced to text alone. Recovering text from pixels avoids parser dependence but requires high-resolution page images and extensive pretraining on text recognition, both of which scale poorly with document length. The role of text accordingly varies across architecture families. Page-to-Document Transformers use text as the primary per-page input, encoded jointly with layout and visual embeddings and exchanged across pages through document-level aggregation \cite{tito2023hierarchical, blau2024gram, borchmann2025arctic}. Backbone-centric MLLMs are typically OCR-free, this is achieved by processing text directly using internal vision capabilities, which is applied to whole-document context in long-context variants \cite{hu2025mplug, yu2024texthawk2} and to retrieved pages or regions selectively in other variants \cite{zheng2026doc, jain2025simpledoc}. Retriever-generators use text as a searchable index over chunks, pages or summaries that determines which evidence reaches the LLM generator \cite{zhang2024cream, shin2025multidocfusion}, while agentic pipelines additionally use text as a navigation layer, where outlines, summaries, and section searches direct later visual inspection \cite{han2025mdocagent, sun2025docagent, wang2025vidorag}.

% \paragraph{OCR Text as Page-Level Representation.} 
% Early hierarchical document transformer models generally treat OCR text as a structured page representation rather than as plain text. Work tokens are paired with bounding boxes, page identifiers, visual patches, and query-conditioned summary or memory tokens. This is so that each page can be encoded locally before document-level aggregation \cite{tito2023hierarchical, dong2024multi}. Later variants add document-global tokens, sparse attention, or layer-wise multimodal fusion to let OCR-based page representations exchange information across pages without fully concatenating all tokens \cite{blau2024gram, borchmann2025arctic}. In this family, OCR text provides the semantic backbone of the page, while layout and visual features preserve grounding.

% % Lewei note: overlap with general MLLM survey, could shorten first half of this paragraph, allowing more space for paragraph above.
% \paragraph{Text as a Visually Latent Skill.}
% OCR-free frameworks often treat reading as an internal LVLM/MLLM capability rather than a separate OCR preprocessing step. This is typically learned by using text recognition, transcription, or grounding as pretraining and instruction-tuning objectives, often with OCR-derived supervision to avoid expensive human annotation. Some models preserve readable text through high-resolution document pretraining and page-level compression \cite{hu2025mplug}, while others strengthen visual reading by aligning text spans with regions and layout cues \cite{yu2024texthawk2}. This skill can also be invoked selectively after retrieving relevant pages, crops, or visual elements \cite{zheng2026doc, jain2025simpledoc}. Visual retrievers may likewise use OCR-derived supervision during training while remaining image-based at inference \cite{tanaka2025vdocrag}. Thus, text becomes a latent skill used where needed, but remains sensitive to image resolution, compression quality, and grounding ability.

% \paragraph{Text for Retrieval and Navigation.}
% Retriever-generator and agentic pipeline models often use extracted text as an index for locating evidence rather than as the full model input. This can involve coarse-to-fine OCR chunk retrieval \cite{zhang2024cream}, page-aware text-image chunks with re-ranking \cite{lopez2025enhancing}, hierarchy-preserving section trees \cite{shin2025multidocfusion}, parent-page or cross-page summary indexes \cite{gong2025mhier}, and generated query nodes over multimodal chunks \cite{zhang2026mldocrag}. In agentic workflows, text also acts as a navigation layer: text and image retrieval can propose complementary evidence \cite{han2025mdocagent}, sparse text retrieval can produce evidence chains for comparison with visual extraction \cite{duan2025m2rag}, and outlines or section search can guide later inspection of pages, tables, and figures \cite{sun2025docagent}. Thus, text is best viewed as a searchable scaffold for multimodal evidence gathering, not a stand-alone document representation.

\subsection{Visual Modality}
Vision in MP-VRDU plays a dual role. Page images carry readable content such as dense text, embedded tables, and equations, and they also carry visual evidence such  as charts, figures, layout cues, and handwriting that cannot be recovered from OCR text alone. Visual token cost scales jointly with page count and per-page resolution, so any encoding scheme must trade page coverage against reading fidelity  within a fixed token budget, and full-document high-resolution ingestion is computationally infeasible for documents of realistic length. Three coarse strategies recur across architecture families. Some systems use low-resolution visual context to keep all pages available at low token cost, with detailed reading provided by another modality such as OCR \cite{tito2023hierarchical, blau2024gram, borchmann2025arctic}. Others perform selective high-resolution reading, applying expensive visual encoding only to pages or regions surfaced by retrieval, navigation, or controller-driven inspection \cite{hu2025mplug, zheng2026doc, jain2025simpledoc}. A third pattern treats vision primarily as a retrieval signal, where visual page embeddings are used to rank evidence rather than to read it in detail \cite{cho2024m3docrag, tanaka2025vdocrag, li2025avir}. Many systems combine more than one of these strategies, for example a low-resolution scan followed by selective high-resolution inspection. Each strategy carries a distinct cost. Low-resolution context loses fine text and small visual elements, selective high-resolution reading inherits the recall ceiling of whichever module decides what to read next, and visual retrieval can rank pages whose layout looks relevant without verifying that the answer-bearing region is actually present, while compression and tokenisation choices further affect how reliably embedded text, table cells, and figure notations survive into the model's input.

\subsection{Layout Modality}
Layout in MP-VRDU operates at two levels. Within a page, it captures spatial relationships such as reading order, columns, table structure, and bounding-box geometry. Across pages, it captures document-level structure such as section hierarchy, page order, multi-page tables, and cross-references between figures, captions, and body text. Within-page layout is largely shared with single-page VRDU and well covered in prior surveys \cite{ding2025survey}, while document-level structure is the more distinctive problem in the multi-page setting. At document scale, per-token spatial annotations such as bounding boxes do not provide an efficient handle on cross-page reasoning, since they describe individual tokens rather than the relations between sections, pages, or referenced elements, and any layout signal must further survive whatever compression, retrieval, or summarisation  step the architecture applies. Three coarse strategies recur. Some systems treat layout as a training objective, encouraging the backbone to preserve spatial and structural information through layout-aware pretraining, masked-region reconstruction, or interleaved text--layout prediction \cite{tito2023hierarchical, hu2025mplug, zhu2025simple}. Others rely on implicit visual layout, where OCR-free MLLMs are expected to recover layout from page pixels using their general visual reading capability \cite{yu2024texthawk2, hu2025mplug}. A third pattern is explicit document structure, where parsed section trees, heading hierarchies, or layout-aware  graph nodes are used to guide retrieval, navigation, or evidence aggregation \cite{shin2025multidocfusion, gong2025mhier, zhang2026mldocrag, wang2024knowledge}. Each strategy carries a distinct cost. Layout-as-objective approaches require additional pretraining or supervision that is expensive to obtain at multi-page scale, implicit visual layout depends on backbone visual capability and on high-resolution input that scales poorly with page count, and explicit document structure depends on parser quality, since errors in heading detection, table-boundary recovery, or reading-order extraction propagate into every downstream retrieval and navigation step.

\subsection{Multimodal Fusion Overview}
Prior MLLM VRDU surveys cover fusion mechanisms such as neural, target-oriented, and prompt-based fusion at length \cite{ding2025survey}. In MP-VRDU, the more distinctive variation is when fusion happens, since full text-vision-layout fusion across every page is computationally infeasible at document scale, so systems differ  on whether modalities meet inside the encoder or only after evidence has been selected. Some systems use early fusion, combining text, visual, and layout features within each page during encoding. Page-to-document transformers do this locally before document-level aggregation \cite{tito2023hierarchical, blau2024gram, borchmann2025arctic}, and backbone-centric MLLMs do it through adapters or projectors that map multimodal features into the LLM token space \cite{tanaka2024instructdoc, jia2024leopard}. Early fusion preserves fine-grained multimodal alignment but is costly when many pages must be processed jointly. Other systems use late fusion, keeping modalities separate during retrieval or inspection and combining them only after candidate evidence has been selected. Retriever-generator pipelines may identify relevant text passages first and bring in visual evidence afterwards \cite{zhang2024cream, gong2025mhier}, and agentic systems extend the pattern by letting separate agents inspect text, tables, and figures before a synthesis or adjudication step \cite{han2025mdocagent, duan2025m2rag, zhu2025doclens}. Late fusion scales better to long documents but depends strongly on retrieval and routing quality. Most multi-page systems sit between these poles, fusing modalities locally within pages and again across selected evidence at the document level.

\paragraph{Parameter efficienct tuning on frozen backbones}
A common strategy is to reuse strong pretrained LLMs or LVLMs and update only a small number of task-specific components. Rather than training vision encoders or language backbones end-to-end, these methods freeze most pretrained parameters and introduce lightweight modules such as LoRA, projectors, adapters, layout tokenizers, or task-specific heads. 
LayTokenLLM~\cite{Zhu2025-jb-laytokenllm} exemplifies this by keeping the LLM backbone frozen and introducing a lightweight layout tokenization module together with LoRA-based adaptation. Rather than modifying the architecture, it injects layout information into the input sequence using a compressed representation and a specialized positional encoding scheme that avoids additional context overhead, enabling efficient scaling to long multi-page documents. As an alternative, some work adopts retrieval-augmented strategies, which avoid directly modeling entire documents by selecting only the most relevant content for downstream reasoning. CREAM~\cite{Zhang2024-zs-cream} introduces a coarse-to-fine retrieval framework. It first retrieves candidate text chunks based on embedding similarity, and then applies multi-round LLM-based re-ranking to identify the most relevant evidence. Extending this idea, MoLoRAG~\cite{Wu2025-mi-molorag} incorporates logical reasoning into retrieval by constructing a graph over document pages and performing multi-hop traversal, enabling the model to capture both semantic and logical relevance across pages.

\paragraph{Curriculum / Multi-Stage Training}
Directly adapting single-page models to multi-page objectives is often ineffective and does not scale well, as it fails to capture cross-page dependencies and long-context interactions.%Either directly adapt single page model, then train on multi-page objectives but doesn't scale well.
Instead, many approaches adopt a progressive training strategy, where models are first trained on simpler local tasks and then gradually extended to more complex multi-page objectives, improving both performance and training efficiency.

A representative example is DocSLM~\cite{Hannan2025-zu}, which adopts multi-stage curriculum. The training begins with image-OCR alignment, followed by stages that introduce image-level and document-level compression, and finally extends to instruction tuning and reasoning. mPLUG-DocOwl2~\cite{Hu2024-wl} follows a related staged training strategy, but with an OCR-free multi-image focus. It first performs single-image structure learning to ensure that compressed visual tokens preserve document text and layout information, then pretrain on multi-page parsing.

\paragraph{Self-Supervised Retrieval Training}

Beyond architectural design, another key challenge lies in effective retrieval mechanisms. Rather than encoding entire documents, many approaches adopt retrieval-augmented pipelines, where models first select relevant pages or segments before performing reasoning. In this setting, the performance of the system is heavily dependent on the quality of the retriever. For example, VDocRAG~\cite{Tanaka2025-er} introduces self-supervised representation compression retrieval that aligns visual document representations with OCR-derived textual content, enabling large vision-language models to learn document embeddings directly from unlabeled document-image pairs.

\section{Datasets}

Datasets in MP-VRDU support pretraining, fine-tuning, and evaluation, and each role increasingly draws on differ
ent sources and supervision. Early work largely shared a small set of multi-page benchmarks for both training and evaluation, including MP-DocVQA with 46,176 questions over 5,928 industrial documents averaging 8.3 pages \cite{tito2023hierarchical}, DUDE with 41,541 questions across 5,019 heterogeneous documents \cite{van2023document}, and SlideVQA with 14,484 questions over 2,600 slide decks of 20 images each \cite{tanaka2023slidevqa}. Pretraining typically draws on broad document archives such as IIT-CDIP, which contains over 6 million scanned business and litigation documents \cite{lewis2006building}, and layout-rich image-text collections such as DocStruct4M with 4 million structure-annotated document images \cite{hu2024mplug}, while genuinely multi-page pretraining corpora remain rare. Fine-tuning has shifted toward model-curated supervision, in which the authors of a specific model assemble corpora from public document sources and transform them through automatic parsing, layout extraction, or LVLM-driven question and trajectory generation to fit that model's training requirements, since per-page annotation does not scale to long documents. Some collections unify prior DocVQA-family supervision into broader skill sets \cite{tanaka2024instructdoc}, while many systems generate synthetic page-grounded question-answer or trajectory data per model \cite{jain2025simpledoc, xiong2026docr1, zhang2026docdancer}. These training distributions are bespoke and rarely released in compatible form, which makes cross-model comparison difficult. Benchmarks have moved from extractive multi-page DocVQA toward long-context multimodal evaluation \cite{ma2024mmlongbench, deng2025longdocurl}, and increasingly score retrieval or page-localisation alongside answer accuracy \cite{hui2024uda, xiong2026docr1, zhu2025doclens}, reflecting the recognition that a correct answer can be produced without correct grounding.

\section{Challenges}

\subsection{Long-Context and Visual Token Scaling}
- No MP-VRDU architecture has decoupled answer quality from document length, only shifting the bottleneck rather than removing it
- Both text and visual modalities scale with page count, a 50-page document exceeds any MLLM backbone's context window
- Each architecture family inherits a distinct failure mode: page-to-document lose fine grained evidence, BC-MLLM attention dilution and lost-in-middle problem, RAG systems bounded by retrieval performance, agentic systems also same retrieval/navigation bottleneck, but also high latency, and tool performance reliability.
